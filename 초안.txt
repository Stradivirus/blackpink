프로젝트 개요 및 기술 스택
이 프로젝트는 기업의 관리자와 고객이 다양한 데이터를 효율적으로 관리하고 시각화할 수 있도록 설계된 통합 웹 플랫폼입니다.
Frontend
React + Vite + TypeScript를 사용하여 컴포넌트 기반 구조로 유지보수와 확장성이 뛰어납니다. 다만 TypeScript를 처음 사용하는 경우 초기 학습 시간이 다소 필요합니다.
Backend
FastAPI를 활용해 RESTful API를 제공합니다. 비동기 처리가 가능해 높은 성능과 빠른 응답속도를 자랑합니다. Flask와 비교했을 때 자동 문서화와 타입 힌트 지원 등의 장점이 있습니다.
DB
MongoDB Atlas를 선택한 이유는 클라우드 기반으로 확장성이 뛰어나며, 개발 도중 스키마 변경이 자유롭고, 팀원들이 로컬에 MongoDB를 설치할 필요가 없어 개발 환경 구성이 간편했습니다.

전체 시스템 아키텍처
아래의 시스템 기능 구조는 프론트엔드, 백엔드, 데이터베이스가 어떻게 연동되는지 한눈에 보여줍니다.
각 기능별로 색상을 지정하여 프론트엔드와 백엔드의 역할 및 주요 기능 영역을 구분하였으며, 영역별 주요 기능과 데이터 흐름, API 연동 구조를 시각적으로 확인할 수 있습니다.
서비스 운영 시, 프론트엔드와 백엔드는 REST API를 통해 통신하며, 백엔드는 MongoDB와 직접 연동하여 데이터를 처리합니다.

프론트엔드 구조 및 주요 기능
대시보드/그래프
관리자와 멤버가 GCI 랭킹, 위험국가 지도, 보안/사업/개발 관련 그래프를 시각적으로 확인할 수 있습니다. 데이터 분석 및 의사결정에 도움을 줍니다.
데이터관리
사업, 개발, 보안 관련 데이터를 CRUD(생성, 조회, 수정, 삭제)할 수 있습니다. 각 데이터는 별도의 엔드포인트에서 관리되며, 실시간으로 백엔드와 연동됩니다.
인증/계정
이 시스템은 외부 회원가입 없이 관리자가 이메일 중복 체크 후, 임시 비밀번호가 포함된 초대 메일을 발송해 회원을 추가하는 방식입니다. 
신규 회원은 메일로 받은 임시 비밀번호로 로그인 후 비밀번호를 변경해 계정을 이용 할 수 있습니다.
이 방식은 보안성과 관리 효율성을 높여주며, 승인된 사용자만 시스템을 이용할 수 있습니다.
게시판
게시글과 댓글을 CRUD할 수 있는 커뮤니티 기능을 제공합니다. 사용자 간 소통 및 정보 공유가 가능합니다.
관리자/멤버
관리자와 멤버의 목록을 조회합니다

백엔드 구조 및 주요 API
관리자/멤버: 멤버 초대, 중복체크, 목록 조회 등 관리 기능을 담당합니다. 각 기능은 별도의 API 엔드포인트로 제공됩니다.
위험국가/지수: 위험국가 지도 데이터와 GCI 랭킹 정보를 제공합니다.
통계/그래프: 보안, 사업, 개발 관련 통계 데이터를 그래프 형태로 제공합니다. 데이터 가공 및 집계 로직이 포함되어 있습니다.
데이터 관리: 사업, 개발, 보안 관련 데이터의 CRUD를 담당합니다. 데이터 무결성과 보안을 위해 인증이 필요합니다.
인증/계정: 로그인, 비밀번호 변경 등 사용자 인증 및 계정 관리 기능을 제공합니다.
게시판: 게시글과 댓글의 CRUD를 위한 API를 제공합니다.

데이터베이스 및 연동
MongoDB Atlas는 클라우드 기반 NoSQL 데이터베이스로, 대용량 데이터 처리와 확장성이 뛰어납니다.
FastAPI 백엔드는 MongoDB와 직접 연동하여 데이터의 저장, 조회, 수정, 삭제를 처리합니다.
데이터 모델은 유연하게 변경할 수 있어 다양한 형태의 데이터를 효율적으로 관리할 수 있습니다.

주요 연동 흐름 및 특징
프론트엔드는 REST API를 통해 백엔드와 통신하며, 실시간 데이터 반영이 가능합니다.
관리자/멤버 관리, 데이터 시각화, 게시판 등 다양한 기능이 통합되어 있어, 사용자는 하나의 플랫폼에서 모든 업무를 처리할 수 있습니다.
컴포넌트 기반 설계와 API 중심 구조로, 새로운 기능 추가 및 유지보수가 용이합니다.
보안, 확장성, 사용자 편의성을 모두 고려한 설계로 실제 서비스 운영에 적합합니다.


권한 관리 시스템

관리자 권한
대시보드/그래프: GCI 랭킹, 위험국가 지도, 각종 통계 그래프 조회 가능
데이터 관리: 사업, 개발, 보안 관련 데이터 CRUD 전체 권한
멤버 관리: 멤버 초대, 목록 조회
게시판: 게시글/댓글 CRUD 전체 권한, 관리자만 공지와 답변 기능 

멤버 권한
게시판: 게시글/댓글 CRUD만 사용 가능, 다른 기능은 접근 제한

인증 시스템
하나의 통합 로그인 함수를 통해 DB의 관리자 컬렉션과 멤버 컬렉션을 동시에 조회하여 사용자 인증을 처리합니다. 
로그인 성공 시 사용자 권한에 따라 접근 가능한 페이지와 기능이 자동으로 제한됩니다.

데이터 처리 최적화
페이지네이션 시스템
데이터의 효율적 처리를 위해 데이터 조회와 게시판에 페이지네이션을 도입했습니다.

향후 개발 계획
기업 정보 조회 시 Comment 컬럼을 추가하여 고객에 대해 알아야 할 점이나 특이 사항 같은 것들을 적어 둘 수 있는 것이 있었으면 좋겠다는 생각이 들었습니다.
세션 보안 기능을 강화하여 중복 로그인 방지, 자동 로그아웃 같은 계정 보안을 강화하면 좋을 것 같습니다.
